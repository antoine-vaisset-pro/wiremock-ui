Fault Response - Network Error Simulation
WireMock can simulate various network failures and errors to test how your application handles error conditions.
Active Fault: {{ selectedMapping.response?.fault }}
Fault Behavior
EMPTY_RESPONSE

The connection will be closed immediately without sending any response data.

  • Socket closes with no HTTP response
  • Client receives connection reset or EOF error
  • Useful for testing connection timeout scenarios
RANDOM_DATA_THEN_CLOSE

Sends random garbage data instead of valid HTTP response, then closes the connection.

  • Sends malformed/corrupted HTTP data
  • Client receives unparseable response
  • Useful for testing response parsing error handling
MALFORMED_RESPONSE_CHUNK

Starts with a valid HTTP 200 response, then sends corrupted data mid-response.

  • Initial HTTP headers appear valid (200 OK)
  • Response body contains malformed chunk encoding
  • Connection closes unexpectedly
  • Useful for testing partial response scenarios
CONNECTION_RESET_BY_PEER

TCP connection is forcefully reset by the server.

  • Simulates abrupt connection termination
  • Client receives "Connection reset by peer" error
  • No HTTP response is sent
  • Useful for testing network failure resilience
